KTaNE Mods

Keep Talking and Nobody Explodes (KTaNE) is a multiplayer co-op game developed by Steel Crate Games about defusing bombs. This is done by solving so-called "modules" which are effectively these small interactive puzzles. The modules that come with the base game include Morse Code, Maze, and Simon Says.

There is a small but active modding community surrounding the game, who contribute modded modules for others to play. These modules are made using the Unity game engine with the C# language. I have made several modded modules of my own, most notably Procedural Maze (top) and Orientation Hypercube (bottom).

Procedural Maze simulates a maze whose walls are generated procedurally based on where the player moves and the current seed, which changes when new walls are generated. The most challenging part of the implementation process was making sure the maze was always able to be solved, since the procedural nature of the maze makes it possible for the player to trap themselves. To do this, I implemented depth-first-search-approach algorithm to attempt solve the maze on generation, and regenerate if unable.

The goal of Orientation Hypercube is to apply rotations to a 4-dimensional hypercube such that it ends up in a target orientation. The most challenging part of the making of this module was creating dynamic meshes to render the 3-dimensional 'faces' of the hypercube. As with Procedural Maze, this module has an autosolver which tries to solve the module in a reasonable amount of moves. In the GIF, it took 11 moves.

Personal Website (this one)

Whilst I have had experience with basic HTML, CSS, and Javascript as a maintainer of the Repository of Manual Pages (below (to be added soon)), I had never built a website before. As this is my very first website, I have decided to forego using any frameworks or libraries other than JQuery.

The most challenging part, for me, was creating a coherent design scheme and keeping everything fairly modular and easily scalable. As is to be expected, the website is not perfect and there are many possible points of improvement. For example, my use of Javascript for loading CSS files and injecting template components such as the navigation bar often causes the website to look a bit janky when loading, and dark mode state is not remembered.

Following on from this project, I would like to make something with the Flask Python framework and learn how to use and interact with databases for future projects.
